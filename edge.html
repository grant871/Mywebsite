Edge/Chrome Updates Galour

Edge and chrome become vulnerable all the time and have zero day threats whereby users can exploit the web browsers code to execute remote code or malicious maleware.

Latest way of updating Edge

[Net.ServicePointManager]::SecurityProtocol = "tls12, tls11, tls"

md -Path $env:temp\edgeinstall -erroraction SilentlyContinue | Out-Null

$Download = join-path $env:temp\edgeinstall MicrosoftEdgeEnterpriseX64.msi

Invoke-WebRequest 'http://go.microsoft.com/fwlink/?LinkID=2093437' -OutFile $Download

Start-Process "$Download" -ArgumentList "/quiet"

Updating Edge fron a standalone installer

cd %homepath%\Desktop

MsiExec.exe /i MicrosoftEdgeEnterpriseX64.msi /qn


The below command opens up the Microsoft edge folder to show what's taking process when Edge is updating

The above images show one edge version taking over another. Usually, if stuck this means a user still has the old version open and will only be updated when the msedge.exe process is close.

Another thing that I have found during my research is that often OS are blocked from reaching the internet e.g. locked down servers or clients that serve specific purposes. Thus there might be a pile up of old versions that have not been able to superseed each other.

The same can be said for Google Chrome and Firefox.
