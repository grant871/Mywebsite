Search the C:\ using Powershell

Get-ChildItem -Path C:\ -Filter "firefox" -Recurse -ErrorAction SilentlyContinue -Force

This can only be used if your on the system - unless you can get something back via intune this script is only useful when you have remoted onto a system.

Removing Folders from a file System

Remove-Item -Path "C:\Program Files\Video Lan" -Recurse -Force

The above script can be used to removed a folder from a windows file system. Why is there a focus for doing this - well if some or a an automated system has not removed a program correctly but a vulnerability detector stil detects the program this is one way we can ensure the .exe is not present to be exploited.

Often used on first time clients

Remove .net 5 EOL

dotnet-core-uninstallremove 5.0.17 --runtime --yes --verbosity q

Remove-Item -Path "C:\Program Files\dotnet\shared\Microsoft.NETCore.App\5.0.17"

The above script can only be used if the .net uninstall tool is install on the remote machines - I have installed the uninstall tool on users machines first then fired this script via intune

Often used on first time clients who wouldn't know .net 5 is installed within their tenant without detection software

Changing and updating registry

Adding New Item - DWord

New-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Services\HTTP\Parameters' -Name "EnableHttp2Cleartext" -Value 0 Force"

New-ItemProperty -Path 'HKLM:\System\CurrentControlSet\Services\HTTP\Parameters' -Name "EnableHttp2Tls" -Value 0 Force"

Everything that is unless specified when injecting into the registry will automatically be a dword and not a string etc..

The above script deters a ddos attack on .net core - it is also updated/patched via a windows update
